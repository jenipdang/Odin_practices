HTML Entities

There are three reserved charcters in HTML: < > and &. You should always use HTML entities for these three characters. Entities always being with an ampersand (&) and end with a semicolon (;).

In between, you put a special code that your browser will interpret as a symbol. In this case, it interprets lt, gt, and amp as less-than, greater-than, and ampersand symbols, respecitively.

Quotes

Four different kinds of curly quotes (opening and closing single and double quotes):

  • “
  • ”
  • ‘
  • ’

If you’re into “web typography,” you’ll also find yourself using curly quotes quite a bit.

UTF-8 and HTML Entities

In the olden days of the web, HTML files weren't allowed to have speical characters in them, making entities much more useful. But, since we're now using UTF-8 character set, we should be able to insert any character directly into the HTML document. This makes entities primarily usefuil as reserved characters or form convenience when authoring stright HTML.
